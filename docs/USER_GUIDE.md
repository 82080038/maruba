## Getting Started

### System Requirements
- Modern web browser (Chrome, Firefox, Safari, Edge)
- Internet connection
- Valid user credentials
- For mobile: Responsive design works on tablets and smartphones

### Login Process
1. Open your browser and navigate to the Maruba URL
2. Enter your username and password
3. Click "Login" button
4. You will be redirected to your role-specific dashboard

### First Time Login
- Change your password immediately after first login
- Update your profile information
- Familiarize yourself with the dashboard layout

## Core Operations

### Member Management

#### Adding New Members
1. Navigate to "Members" â†’ "Add Member"
2. Fill in required information:
   - Personal details (name, address, phone)
   - Identification (NIK, KK)
   - Contact information
   - Membership category
3. Upload required documents
4. Review and submit
5. Member account is created automatically

#### Updating Member Information
1. Search for member using search bar
2. Click "Edit" on member record
3. Update necessary fields
4. Save changes
5. System logs all modifications

#### Member Status Management
- **Active**: Full member privileges
- **Inactive**: Limited access
- **Suspended**: No access
- **Withdrawn**: Closed membership

### Loan Management

#### Loan Application Process
1. Member submits loan application
2. Application is reviewed by Manajer
3. Credit assessment is performed
4. Loan is approved/rejected
5. Disbursement is processed (if approved)

#### Loan Types
- **Pinjaman Biasa**: Regular loans
- **Pinjaman Darurat**: Emergency loans
- **Pinjaman Produktif**: Business loans
- **Pinjaman Konsumtif**: Consumer loans

#### Loan Disbursement
1. Approved loan appears in disbursement queue
2. Kasir processes disbursement
3. Funds are transferred to member
4. Receipt is generated
5. Loan repayment schedule is created

#### Loan Repayment
1. Member makes payment
2. Kasir records payment
3. System updates loan status
4. Receipt is issued
5. Payment history is updated

### Savings Management

#### Savings Types
- **Simpanan Pokok**: Initial deposit
- **Simpanan Wajib**: Monthly mandatory savings
- **Simpanan Sukarela**: Voluntary savings

#### Deposit Process
1. Member requests deposit
2. Teller processes transaction
3. Amount is credited to account
4. Receipt is generated
5. Account balance is updated

#### Withdrawal Process
1. Member requests withdrawal
2. Teller verifies balance
3. Withdrawal is processed
4. Receipt is generated
5. Account balance is updated

## Troubleshooting

### Common Issues

#### Login Problems
- **Issue**: Cannot login
- **Solution**: Check username/password, contact admin
- **Issue**: Account locked
- **Solution**: Wait 30 minutes or contact admin

#### Transaction Issues
- **Issue**: Transaction not processing
- **Solution**: Check internet connection, try again
- **Issue**: Payment gateway error
- **Solution**: Check payment method, use alternative

#### Performance Issues
- **Issue**: Slow response time
- **Solution**: Clear browser cache, check internet speed
- **Issue**: Page not loading
- **Solution**: Refresh page, check browser compatibility

### Error Messages

#### Common Error Messages
- "Invalid credentials": Username or password incorrect
- "Access denied": Insufficient permissions
- "Transaction failed": Payment processing error
- "Connection timeout": Network issue

#### Error Resolution Steps
1. Note the exact error message
2. Check internet connection
3. Clear browser cache
4. Try again
5. Contact support if issue persists
